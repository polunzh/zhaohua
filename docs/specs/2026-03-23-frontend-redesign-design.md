# 朝花夕拾 — 前端重做设计

## 概述

将当前的纯色占位 Canvas 前端重做为像素风复古 RPG 游戏界面。

## 设计决策

### 整体风格

像素风复古 RPG，俯视角。

### 布局

左侧信息栏 + 中央像素场景 + 底部对话栏（方案 B）。

- 左栏：时间/天气、事件日志、导航（可去的地方）
- 中央：Tile Map 像素场景，NPC 在其中走动
- 底部：对话框 + 选项按钮

### 画面色调

褪色暖调像素风。用设计文档定义的旧时光色板做像素画：低饱和、暖黄底、暗角、微颗粒感。像一张泛黄的老照片变成了像素画。

### 场景实现

Tile Map 系统：

- 定义 tile 图块集（地面、墙壁、屋顶、桌椅、树木、花草等）
- 场景用二维数组拼出
- 分层渲染：地面层 → 物体层 → 角色层 → 天气层
- 季节变化换调色板
- 素材原创或使用 CC0/MIT 开源许可，不侵权

### NPC

24×32 大尺寸像素 sprite：

- 有发型、衣服颜色等辨识细节
- 名字旁带 emoji 显示当前心情
- 点击触发对话
- 在场景中按日程走动

### 校园布局（俯视图）

- 上方：操场（升旗台、篮球架）
- 中央：教学楼（一排平房，左教室右办公室，中间大门）
- 左侧：花池（缩小版，铺满月季、黄花、紫花、竹子，密密麻麻看不到地面）
- 右侧：水塔（高水塔 + 水龙头，夏天学生排队）
- 下方：校门 + 土路（通向村子）
- 四周：散落的树

走到建筑门口可进入室内（独立 tile map），走到校门切换到村路场景。

## 技术方案

### Tile Map 引擎

- Canvas 渲染，每个 tile 16×16 像素
- 地图数据：二维数组，每个元素是 tile ID
- 多层渲染：ground、objects、characters、weather
- 碰撞检测：标记哪些 tile 可走、哪些不可走
- 视口跟随玩家角色移动

### Tile 素材

- 用代码生成基础 tile（像素绘制到小 canvas 再导出）
- 或使用开源像素素材库（CC0 许可）
- tile 集统一管理在一个 tileset 图片中

### 角色 Sprite

- 24×32 像素，4 方向（上下左右）× 3 帧走动动画
- 每个角色有不同的衣服颜色和发型
- Sprite sheet 统一管理

### 滤镜叠加

- 渲染完成后对整个 canvas 应用旧时光滤镜（已有 applyColorGrading）
- 暗角、微颗粒感

### UI 组件重构

- 左栏：Vue 组件，不在 Canvas 内
- 对话框：Vue 组件，覆盖在 Canvas 下方
- 选项按钮：在对话框内
- 地图 Canvas：居中显示

### 需要替换的现有代码

- `src/scene/renderer.ts` → 重写为 Tile Map 渲染器
- `src/scene/scenes.ts` → 重写为 tile map 数据格式
- `src/components/GameCanvas.vue` → 适配新渲染器
- `src/App.vue` → 新布局
- 保留：`src/scene/filter.ts`（旧时光滤镜）
