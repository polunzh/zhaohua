# 朝花夕拾 — 设计文档

## 概述

一款基于 Web 的生活模拟体验，让玩家在 1994-2002 年中国北方农村的世界里，体验小学老师和邮递员的日常生活。世界时间与现实同步，24 小时运转。玩家随时打开浏览器就能探进这个小世界，看看此刻正在发生什么。

这不是挑战型游戏，没有输赢。是一段平静的、有烟火气的旧时光体验。

## 开发原则

先做出来，再做好。MVP 阶段只做最小可用的核心体验，复杂功能后面慢慢加。克制，不过度设计。

## 一、世界观与设定

**时间：** 游戏内时间跨度为 1994-2002 年，从某个秋季开学开始。现实时间与游戏内时间的映射：现实 1 天 = 游戏内 1 天（一天内的时刻同步），但游戏内日历可以加速推进——玩家可以选择"跳到下一天/下周/下个学期"来加速时间流逝，避免需要现实 8 年才能走完故事线。默认按实时走，加速是可选操作。

**地点：** 中国北方某虚构村子和镇子。村子有小学、几十户人家、农田、一条土路通向镇上。镇上有邮局、集市、卫生所。

**两条线：**
- **老师线：** 村小学老师，教几个年级的课（农村小学常一人教多科）。日常是备课、上课、批作业、和学生/家长打交道。
- **邮递员线：** 镇上邮局的邮递员，骑自行车穿梭在镇和周边几个村之间，送报纸、信件、偶尔有包裹。

两条线在同一个世界里交织。邮递员会来学校送东西，老师可能托邮递员带信。村民、学生是共享的 NPC。

**两条线的切换：** 玩家可以随时切换操控老师或邮递员。未被操控的角色按 NPC 逻辑自动推进（老师按课表上课，邮递员按路线送信）。切换后可以看到对方在你不操控期间做了什么。

**世界归属：** 单部署单玩家。整套系统为部署者一个人服务，不涉及多用户、账号体系或数据隔离。

**四季与环境：**
- **春天：** 冰雪消融，柳树发芽，油菜花开，偶尔倒春寒。农忙开始，有学生请假
- **夏天：** 很热，校园花园花开，知了叫，暑假前的期末考试。水塔前学生排队喝水
- **秋天：** 开学季，收玉米收花生，天渐凉，树叶变黄
- **冬天：** 很冷，教室生煤炉，学生手上有冻疮，路上有冰，快到年底有年味

**氛围基调：** 平静、温暖、带点苦涩。不美化也不卖惨，是真实的、有烟火气的农村日常。

## 二、核心玩法循环

**打开浏览器时发生什么：**

1. **追算时间差** — 系统算出上次关闭到现在过了多久，快速模拟这段时间里世界发生了什么（天气变化、NPC 活动、事件触发）
2. **呈现当前时刻** — 一幅手绘插画风的场景画面，对应现实时间。比如现实早上 7 点，看到的是村小学的早晨，学生沿着土路走来上学
3. **回顾摘要** — 如果错过了一些事，左侧有简短的"这段时间发生了什么"（比如"昨天王大爷收到了儿子从城里寄来的信"）
4. **互动** — 点击场景中的人物/物品触发对话和事件，关键时刻弹出选项

**日常节奏（老师线）：**
- 早晨：学生沿土路步行到校，有的结伴走，升旗，早读
- 上午/下午：上课、批作业、处理学生的小事
- 课间：和学生聊天、在花池散步、水塔旁看学生排队喝水打闹
- 放学后：家访、备课、和同事聊天

**日常节奏（邮递员线）：**
- 早晨：去邮局取当天的报纸和信件
- 上午：骑车出发，按路线送到各个村。路上可能遇到走路上学的学生
- 途中：和村民打招呼、遇到各种小状况
- 下午：回镇上，整理明天的东西

**交互方式：**
- 场景可点击探索（人物、物品、场景元素）
- 关键时刻弹出选项（"去教室"、"去送下一家的信"、"帮学生补课"）
- 两者结合，自由探索 + 选择推动

**体验节奏：**
- 每次打开几分钟即可
- 游戏时间与现实时间同步，世界 24 小时运转
- 不玩的时候世界继续走，下次打开时可以看到错过的事

## 三、技术架构

**整体结构：** 前后端分离的 Web 应用。

**工具链：** VitePlus（dev/build/test/check），基于 Vite + Rolldown + Vitest + Oxc。

**前端：**
- Vue 3
- Canvas 渲染手绘风场景（预绘制的场景底图 + 动态叠加 NPC/物品图层）
- 场景素材：按季节、时间段、天气组合，每个场景有若干可点击区域
- 交互层：点击人物/物品触发对话框或选项面板
- Vitest 做测试，Oxlint/Oxfmt 做代码检查和格式化

**后端：**
- Nitro（VitePlus 生态的后端方案）
- 核心职责：世界状态管理、时间追算、事件生成、调用 AI API 生成对话
- 不需要 24 小时运行，只在用户访问时启动计算

**时间追算引擎：**
- 用户打开页面时，取上次关闭时间和当前时间的差值
- 按 15 分钟为步长模拟：每步检查事件触发条件（天气变化、NPC 位置变动、随机事件），确保课间、到校、送信途中等细粒度场景能被正确匹配
- 生成一份"错过的事件摘要"展示给用户
- 当前时刻的场景完整渲染
- 支持"跳到下一天/下周/下学期"的加速操作，跳过时批量追算
- 追算性能约束：离线超过 7 天（672 步）时，切换为"天级"粗粒度追算（只计算每天的关键事件，跳过日常琐事），避免上万次 tick 阻塞页面加载。摘要最多展示最近 20 条重要事件
- 跳时锚点规则：执行"跳到下一天"后，游戏内日历前进到下一天同一时刻，现实时间重新锚定为当前时刻。之后继续按实时同步走。本质上是维护一个 `calendar_offset`（游戏日历与现实日历的偏移量），每次跳时增加偏移
- 并发访问：追算过程对世界状态加写锁（SQLite 事务），同一时刻只允许一个追算过程执行。第二个标签页/设备访问时等待锁释放后读取已追算的状态，不会重复推进

**AI 多模型支持：**
- 统一的 AI 接口层，上层调用不关心具体模型
- 适配器模式，每个模型一个适配器：Kimi、DeepSeek、MiniMax、Claude、OpenAI 等
- 模型选择和 API Key 在后端 .env 中配置（单人私有部署，不需要前端设置页）
- 后续新增模型只需加一个适配器
- AI 的边界：世界状态转移（NPC 位置、事件触发、天气变化、日历推进）由确定性规则引擎驱动，不依赖 AI。AI 只负责文本表达层——对话润色、事件描述文案、摘要生成。这样保证不同模型、不同版本下世界状态一致，AI 切换只影响文字风格
- 用途：对话生成（NPC 说话内容，带时代和地域特色）、事件描述文案、错过事件摘要、长线故事的叙述文本

**数据存储（SQLite）：**
- 世界状态：当前游戏内日期时间、天气、季节、上次访问时间
- NPC 状态：每个角色的位置、心情、与玩家的亲密度、日程表
- 事件日志：发生过的事件记录（时间戳、类型、涉及角色、结果），支持错过事件摘要回溯
- 玩家选择历史：用于影响后续故事走向
- 长线故事进度：每条故事线的当前阶段、关键节点、因果关系
- 信件/包裹追踪：投递状态、来源、目的地

**配置存储：**
- AI 模型选择和 API Key 统一在后端 .env 中配置，部署者直接编辑，不暴露在前端，不需要前端设置页

## 四、NPC 与角色系统

**核心 NPC（老师线）：**
- 学生们：十几个孩子，不同年级混在一起，各有性格。有调皮的、安静的、成绩好的、经常迟到的
- 校长：一个人撑起整个学校的老校长
- 家长们：农忙时见不到人，农闲时会来学校问孩子情况
- 同事：一两个老师，大家什么都教

**核心 NPC（邮递员线）：**
- 邮局同事：镇上邮局的人
- 村民们：各村收信的人，有的盼远方亲人的信，有的订了报纸
- 小卖部老板：路上歇脚的地方

**共享 NPC：**
- 邮递员和老师互相认识，邮递员送信到学校
- 村里的孩子既是学生也是邮递员路上遇到的人

**角色状态：**
- 每个 NPC 有基本属性：心情、与玩家的亲密度、当前位置
- NPC 有自己的日程表（学生几点上学放学，村民几点下地干活）
- 关系会因玩家选择而变化（比如帮某个学生补了课，关系更好）

**对话特色：**
- 带时代感的语言，不用现代网络用语
- 农村口语化，朴实直接
- AI 生成时提供角色性格和背景作为 prompt context

## 五、美术与场景系统

**场景组织：**
- 按地点划分：学校（教室、操场、花池、水塔、办公室）、村子（土路、农田、人家门口）、镇上（邮局、集市、卫生所）、路上（村与村之间的路）
- 每个地点有四季 × 时间段（早/午/晚/夜）的变体
- 天气叠加层：晴、阴、雨、雪

**学校特色场景：**
- 花池：30×40 米，种着月季花、竹子等植物。课间散步、和学生聊天的重要场所。春天月季发芽，夏天花开得热闹，秋天渐落，冬天只剩竹子还绿着
- 水塔：同学们排队接水喝的地方，夏天尤其热闹，下课铃一响就有一群孩子跑去排队，是学生之间闲聊、打闹的社交场所

**旧时光视觉风格：**
- 参考 90 年代柯达 Gold 200 / 富士 Superia 胶片的色彩感觉——偏暖、低饱和、阴影发黄
- 色调控制：最高饱和度不超过 35-40%，黑色用暖棕（`#3A3530`）而非纯黑，白色用泛黄（`#F2E8D5`）而非纯白
- 光线效果：柔和的暖色暗角、微弱的胶片颗粒感、高光带轻微泛黄光晕
- 风格参考：90 年代人教版语文课本插图（水彩、柔和轮廓线）、连环画（线描 + 平涂）
- 纸张质感：冷压水彩纸或新闻纸的纹理底层
- 字体：思源宋体，贴合年代感
- 整体感受：像翻开一本旧相册，克制、微妙，不是 Instagram 滤镜

**主色板：**

| 用途 | 色值 | 描述 |
|------|------|------|
| 底色/旧纸 | `#F5E6C8` | 泛黄的纸张底色 |
| 天空 | `#A8B8B0` | 低饱和青灰，农村阴天的天 |
| 褪色红 | `#C4706A` | 春联、红旗的褪色红 |
| 旧绿 | `#7A9178` | 老课本封面绿，庄稼田 |
| 暖影 | `#6B5B4E` | 棕调阴影 |
| 麦黄 | `#D4C08E` | 晒褪色的麦田/土路 |
| 墨蓝灰 | `#5C6B7A` | 褪色圆珠笔/油墨色 |
| 肤色/土色 | `#C9A882` | 暖土和肤色中间调 |

**动态元素：**
- 场景底图是静态手绘图
- 上面叠加可动图层：NPC 出现/离开、烟囱冒烟、树叶飘动、知了声等简单动效
- 点击高亮区域时有轻微视觉反馈

**音效（后期可选）：**
- 环境音：鸟叫、风声、知了、下雨、煤炉噼啪
- 不加背景音乐，安静更有味道

**素材策略：**
- 初期用 AI 生成插画作为底图，人工调整保持风格统一
- NPC 用几套基础立绘 + 配色变化来区分
- 先做核心场景（教室、土路、邮局），逐步扩展

## 六、长线故事与事件系统

**事件分层：**
- **日常琐事（每天都有）：** 学生迟到、作业没写、邮件分拣错了、路上遇到村民闲聊
- **小事件（隔几天一次）：** 学生打架、家长来访、一封寄错地址的信、自行车坏了、下雨路滑
- **季节事件（固定节点）：** 开学、期中考试、期末考试、放假、农忙、过年、春游
- **长线故事（跨周甚至跨月）：** 一个学生从厌学到慢慢喜欢上课、一户人家等了很久终于收到远方的信、校长争取经费修教室

**故事生成机制：**
- 世界状态推进由确定性规则引擎驱动：事件池 + 触发条件 + 随机种子，保证可复现
- 日常琐事和小事件从事件池中按条件抽取，AI 负责润色描述文案
- 季节事件按游戏内日历自动触发
- 长线故事预定义阶段和分支节点，规则引擎根据角色状态和玩家选择推进到下一阶段，AI 负责生成叙述文本
- 玩家的选择会影响后续走向，但不会有"失败"结局——这不是挑战型游戏，是生活体验

**基调把控：**
- 不美化也不卖惨，真实平静
- 有温暖的瞬间，也有无奈的事（比如好学生因为家里穷要辍学）
- 节奏慢，留白多，不催促玩家做任何事
